
## 已确认需求

### 平台
- Web（PC + 手机浏览器响应式），后期扩展微信小程序

### 分期
- **一期角色：** 督导、店员、系统管理员
- **二期角色：** 营销总监、片区经理（功能含目标下发、数据看板、复盘查阅）
- 一期督导**自设**月度目标，无需上级下发

### 登录 & 账号
- 飞书 OAuth 扫码登录（罗莱已全面使用飞书）
- 同步飞书通讯录组织架构，免手动录入层级
- 营销总监/片区经理登录后一期显示「功能即将开放」占位页

### PDCA 核心流程

| 阶段 | 操作者 | 内容 |
|------|--------|------|
| P（计划） | 片区经理 → 督导 | 月目标逐级下发，督导拆解到门店/店员，形成周目标和日目标 |
| D（执行） | 店员 | 每天下班后填写：实际销售额 + 未达成原因 |
| C（检查） | 系统自动 | 红绿状态自动触发，聚焦未完成人员 |
| A（处理） | 督导 + AI | 周复盘：AI 自动生成周总结文字 + 数据图表 |

### 排名功能
- 按周展示 TOP10：店员 / 门店 / 督导
- 排名维度：销售额绝对值 + 目标完成率，可切换
- TOP1 店员风采半身照（手动上传）展示在排名看板旁

### 经验推送功能
- 周总结中优秀 TOP10 的销售经验，管理员可推送至首页
- 产品定位：兼具激励、竞争、经验传播的销售作战平台

### 各角色使用场景

| 角色 | 主要场景 |
|------|----------|
| 营销总监/片区经理 | 快速扫视全局数据，下钻查看 |
| 督导 | 看排名 + 看预警 + 发指令 |
| 店员 | 填报 + PK 排名 + 看经验推送 |

### 数据模型（核心表）

| 表 | 说明 |
|---|---|
| `users` | 用户，含角色 + `avatar_url` + `profile_photo_url`（风采照） |
| `stores` | 门店，归属督导和片区 |
| `regions` | 片区 |
| `targets` | 月/周/日目标，按用户+门店+时间维度 |
| `daily_reports` | 店员每日填报（实际销售额 + 未达成原因） |
| `weekly_reviews` | AI 生成的周总结（文字 + 图表数据） |
